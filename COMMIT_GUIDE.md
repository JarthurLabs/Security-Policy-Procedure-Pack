# Commit Guide

Use commit messages that describe the security outcome or governance improvement.

## Recommended Commit Messages

```text
Add IAM governance workflow documentation
Create security program traceability matrix
Add quarterly access review process
Add privileged account approval workflow
Add MFA enforcement matrix
Add customer security questionnaire response pack
Add Google Workspace security validation checklist
Add 30-60-90 security remediation roadmap
Add security metrics and KPI definitions
Update policy pack with ownership and review schedule
```

## Avoid

```text
Add files via upload
Update README
misc changes
final version
new stuff
```

## Why This Matters

Professional commit messages make the repository easier to review. They show organized work, iteration, and clear intent.
