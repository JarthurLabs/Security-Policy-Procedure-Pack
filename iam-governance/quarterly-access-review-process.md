# Quarterly Access Review Process

## Purpose

Quarterly access reviews verify that user access still matches current job responsibilities.

## Review Scope

At minimum, review:

- Admin accounts
- Finance and billing systems
- Customer data systems
- Shared drives containing sensitive data
- Password manager groups
- Contractor accounts
- Inactive users
- External collaborators

## Review Steps

| Step | Owner | Evidence |
|---|---|---|
| Export current user access | IT administrator | Access export |
| Identify privileged users | IT administrator | Admin user list |
| Send access list to system owner | Operations manager | Review request |
| Confirm valid access | System owner | Signed review response |
| Remove unnecessary access | IT administrator | Removal evidence |
| Document exceptions | Operations manager | Exception record |
| Save final review package | Operations manager | Review folder |

## Review Decisions

| Decision | Meaning |
|---|---|
| Keep | Access still matches job need |
| Remove | Access is no longer required |
| Modify | Access should be reduced or changed |
| Exception | Access does not meet standard but is temporarily approved |
| Investigate | Reviewer does not recognize access |

## Review Frequency

| Access Type | Frequency |
|---|---|
| Standard application access | Quarterly |
| Admin access | Quarterly |
| Contractor access | Monthly |
| Shared external access | Quarterly |
| Finance/billing access | Quarterly |

## Success Criteria

A quarterly access review is complete when:

1. All in-scope access has an owner response.
2. Unneeded access is removed.
3. Exceptions are documented with expiration dates.
4. Evidence is saved for future review.
