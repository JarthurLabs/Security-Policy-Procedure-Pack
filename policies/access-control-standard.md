# Access Control Standard

## Document Owner

Operations Manager

## Purpose

This standard defines how access is requested, approved, reviewed, changed, and removed.

## Scope

This standard applies to all business applications, SaaS platforms, shared drives, finance systems, customer data systems, and administrative accounts.

## Principles

### Least Privilege

Users should only receive the access needed to perform their job duties.

### Role-Based Access

Access should be assigned based on job role when possible.

### Named Accounts

Shared accounts should be avoided because they reduce accountability.

## Access Request Process

1. Manager submits access request.
2. System owner reviews business need.
3. IT admin grants the minimum required access.
4. Access is documented in the access tracker.
5. User receives basic handling expectations.

## Offboarding Process

When a user leaves the company:

1. Disable account access the same day.
2. Revoke SaaS application access.
3. Transfer file ownership if needed.
4. Remove shared drive permissions.
5. Record completion in the offboarding checklist.

## Access Review

Access should be reviewed quarterly for:

- Admin accounts
- Finance systems
- Customer data systems
- Contractors
- Inactive users

## Evidence

Evidence may include:

- Access review export
- Offboarding checklist
- Approval ticket
- Manager approval
- Screenshot of permission removal
