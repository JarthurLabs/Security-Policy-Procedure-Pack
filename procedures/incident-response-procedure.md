# Incident Response Procedure

## Owner

Operations Manager

## Purpose

Define a repeatable workflow for responding to suspected cybersecurity incidents.

## Response Workflow

1. Identify the issue.
2. Record who reported it and when.
3. Preserve evidence.
4. Contain affected account, device, or system.
5. Investigate available logs and user activity.
6. Remove the cause.
7. Restore normal operations.
8. Document lessons learned.

## Severity Levels

| Severity | Description | Example |
|---|---|---|
| Low | Limited issue, no sensitive data | Spam email reported |
| Medium | Possible account or data exposure | Suspicious login alert |
| High | Confirmed unauthorized access | Compromised email account |
| Critical | Major disruption or widespread data exposure | Ransomware |

## Evidence to Preserve

- Screenshots
- Login timestamps
- IP addresses
- Email headers
- User reports
- Admin logs
- Vendor notices
