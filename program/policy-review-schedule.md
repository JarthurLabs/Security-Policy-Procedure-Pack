# Policy Review Schedule

## Purpose

Security documents must be reviewed regularly so they remain accurate and useful.

| Document | Owner | Review Frequency | Next Review Trigger |
|---|---|---|---|
| Acceptable Use Policy | Operations Manager | Annual | Employee behavior incident or major system change |
| Password and MFA Standard | IT Administrator | Semiannual | New critical SaaS system or authentication incident |
| Access Control Standard | Operations Manager | Quarterly | Access review findings or role changes |
| Vendor Risk Standard | Business Owner | Annual | New vendor category or customer requirement |
| Incident Response Procedure | Operations Manager | Semiannual | Incident or tabletop exercise |
| Customer Questionnaire Response Pack | Operations Manager | Quarterly | New customer security request |
| Security Metrics and KPIs | Operations Manager | Monthly | Leadership review |

## Review Evidence

Each review should record:

- Date reviewed
- Reviewer
- Changes made
- Approval status
- Next review date
