# Security Ownership Chart

## Purpose

Define who owns security decisions and supporting evidence.

| Security Area | Primary Owner | Backup Owner | Responsibilities |
|---|---|---|---|
| Access governance | Operations Manager | IT Administrator | Access reviews, approvals, exceptions |
| MFA and admin settings | IT Administrator | Operations Manager | MFA enforcement, admin access, technical validation |
| Vendor risk | Business Owner | Operations Manager | Vendor review, risk scoring, approval tracking |
| Incident response | Operations Manager | IT Administrator | Triage, containment coordination, incident log |
| Customer assurance | Operations Manager | Business Owner | Questionnaire responses, evidence references |
| Policy governance | Operations Manager | Compliance Lead | Policy review schedule and updates |
| Employee awareness | Operations Manager | IT Administrator | Awareness guidance and acknowledgement tracking |

## RACI Snapshot

| Activity | Responsible | Accountable | Consulted | Informed |
|---|---|---|---|---|
| Approve standard access | Manager | System Owner | IT Admin | Operations Manager |
| Approve privileged access | System Owner | Operations Manager | IT Admin | Manager |
| Remove leaver access | IT Admin | Operations Manager | Manager | System Owner |
| Approve vendor | Business Owner | Operations Manager | IT Admin | Leadership |
| Respond to incident | Operations Manager | Leadership | IT Admin | Affected stakeholders |
