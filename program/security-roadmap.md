# Security Roadmap

## Purpose

This roadmap organizes security improvements into practical phases for a small business.

## Phase 1: Stabilize Access and Authentication

| Initiative | Owner | Priority | Target |
|---|---|---|---|
| Enforce MFA for email and admin accounts | IT Administrator | Critical | 30 days |
| Create access request workflow | Operations Manager | High | 30 days |
| Create offboarding checklist | Operations Manager | High | 30 days |
| Identify all admin users | IT Administrator | High | 30 days |

## Phase 2: Improve Governance and Evidence

| Initiative | Owner | Priority | Target |
|---|---|---|---|
| Complete first quarterly access review | Operations Manager | High | 60 days |
| Implement vendor risk checklist | Business Owner | Medium | 60 days |
| Define security KPIs | Operations Manager | Medium | 60 days |
| Create incident log template | Operations Manager | Medium | 60 days |

## Phase 3: Validate and Mature

| Initiative | Owner | Priority | Target |
|---|---|---|---|
| Complete Google Workspace security audit | IT Administrator | High | 90 days |
| Run incident response tabletop exercise | Operations Manager | Medium | 90 days |
| Review customer questionnaire evidence | Operations Manager | Medium | 90 days |
| Report security metrics to leadership | Operations Manager | Medium | 90 days |
