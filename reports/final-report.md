# Final Report: Small Business Security Policy & Procedure Pack

## Executive Summary

This project created a practical security documentation package for a simulated small business, then strengthened it with evidence of decision-making. The final package includes baseline policies, an incident response procedure, access control guidance, vendor risk review, risk scoring, control mapping, before/after maturity analysis, and a changelog.

The project is designed to demonstrate beginner-level GRC, security implementation, and cybersecurity documentation capability.

## Business Need

Northstar Dental Billing LLC needed a lightweight cybersecurity governance baseline. The company did not require a full compliance program, but it needed practical documentation that could improve internal behavior and support customer trust.

## Work Completed

| Workstream | Output |
|---|---|
| Governance | Acceptable use policy, password/MFA standard, access control standard |
| Incident Response | Incident response procedure with severity levels and evidence guidance |
| Vendor Risk | Vendor review checklist and scorecard |
| Risk Management | Risk register with likelihood x impact scoring |
| Framework Alignment | NIST CSF 2.0 and CIS Controls mapping |
| Portfolio Evidence | Screenshots, changelog, before/after analysis |

## Key Decisions

### Decision 1: Focus on Access and Authentication First

The highest-rated risks were related to access and authentication because account compromise can lead directly to customer data exposure, business disruption, and incident response costs.

### Decision 2: Use Lightweight GRC Instead of Overbuilding

The simulated company is small, so the documents were designed to be practical and readable. The goal was not to create enterprise-heavy policies that no one would follow.

### Decision 3: Add Vendor Review

Small companies often adopt SaaS tools quickly. Adding vendor risk review makes the project more realistic and relevant to modern security operations.
