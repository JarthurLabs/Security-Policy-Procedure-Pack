# Final Report

## Summary

This project models a small-business security governance program. The strongest parts are IAM governance, vendor risk review, customer assurance support, and the traceability matrix that links risks to policies, controls, procedures, owners, and status.

This is not meant to look like a fully mature compliance program. It is a realistic starting point for a small company that needs clearer security ownership and better evidence.

## Initial observations

The main issue was not that the company had no security ideas. The issue was that most security decisions were informal. Access approvals, offboarding, vendor review, and customer questionnaire responses all depended too much on memory and individual judgment.

## Important gaps

- There is no live system export.
- No real vendor contract or SOC 2 report was reviewed.
- The technical validation checklist is simulated.
- No training completion records exist.
- No tabletop exercise was performed.

## Practical next steps

1. Test the offboarding checklist with a mock user.
2. Add a real access export from a lab SaaS environment.
3. Track one quarterly access review from start to finish.
4. Run a simple incident response tabletop.
5. Add evidence screenshots from a real test Google Workspace or Microsoft 365 tenant.

## Reflection

This project helped clarify that GRC work is not just writing policies. The more realistic work is connecting policy, ownership, evidence, and follow-up.
