# CIS-Lite Endpoint Hardening Checklist

## Purpose

This checklist provides a lightweight endpoint hardening validation example inspired by common CIS Benchmark concepts. It is not a full CIS Benchmark assessment.

## Checklist

| Check | Expected State | Evidence | Status |
|---|---|---|---|
| Automatic updates enabled | Enabled | Screenshot or MDM export | Planned |
| Disk encryption enabled | Enabled | Device settings screenshot | Planned |
| Screen lock configured | 15 minutes or less | Device policy screenshot | Planned |
| Local admin accounts limited | Only approved admins | User/group export | Planned |
| Firewall enabled | Enabled | Device security settings | Planned |
| Antivirus/EDR active | Active | Security tool dashboard | Planned |
| Browser auto-update enabled | Enabled | Browser settings or MDM profile | Planned |
| Unapproved extensions reviewed | Reviewed monthly | Extension inventory | Planned |
| Device inventory maintained | Current asset list | Asset inventory | Planned |

## Note

This is included to demonstrate how governance can connect to endpoint configuration validation. It should not be represented as a full technical benchmark audit.
